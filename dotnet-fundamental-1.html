3) Value vs Reference Types

Value Types int, double, bool, struct. Copy = new value.

int x = 5; int y = x; y++; // x still 5

Reference Types class, string, array. Copy = same reference.

var a1 = new int[]{1,2}; var a2 = a1; a2[0] = 99; // a1[0] == 99

4) Boxing & Unboxing

int a = 42;
object boxed = a;   // boxing
int b = (int)boxed; // unboxing

Avoid unintended boxing in hot paths (old collections like ArrayList, interface calls on value types, etc.).

5) Strings & Immutability

string s = "Hello"; s += " World"; // new string allocated
// Use StringBuilder for heavy concatenation

6) Operators: Null-Safety & Switch Expressions

Null safety

string? input = null;
string name = input ?? "Guest";  // coalescing
name ??= "Guest";                 // assignment
int? len = name?.Length;          // conditional access

Switch expressions & patterns

string Category(int n) => n switch {
  < 0   => "neg",
  0     => "zero",
  <= 10 => "small",
  <= 100=> "medium",
  _     => "large"
};

7) var, dynamic, object & Defaults

var i = 42;          // compile-time inference
object o = 42;       // boxed int
dynamic d = "hello"; // runtime-typed
Console.WriteLine(default(int));    // 0
Console.WriteLine(default(string)); // null

Guideline: Prefer var when type is obvious; avoid dynamic unless required.

8) const / readonly / static readonly

public class Config {
  public const double Pi = 3.14159;     // compile-time const
  public readonly Guid Id = Guid.NewGuid(); // set in ctor
  public static readonly DateTime Start = DateTime.Now; // per-type
}

9) Expressions vs Statements & Expression-bodied

public class Person {
  public string Name { get; }
  public Person(string name) => Name = name; // expression-bodied ctor
  public override string ToString() => $"Name: {Name}";
}

10) Tuples, Deconstruction & Discards

var person = (Name: "Sonny", Age: 28);
var (name, age) = person;
var (_, onlyAge) = person; // discard first

11) Records (preview for OOP)

public record Person(string First, string Last);
var p1 = new Person("Sonny","Bui");
var p2 = p1 with { Last = "Nguyen" };

Immutable by default with value-based equality.

12) Type Conversions & TryParse

double x = 3.14; int n = (int)x; // explicit
if (int.TryParse("123", out var v)) { /* use v */ }

Prefer culture-invariant parsing for CLI tools:

double.TryParse(text, System.Globalization.NumberStyles.Float,
  System.Globalization.CultureInfo.InvariantCulture, out var value);

13) checked / unchecked

checked {
  int a = int.MaxValue;
  int b = a + 1; // throws OverflowException
}

14) Local Functions

int Factorial(int n) {
  return Inner(n);
  int Inner(int k) => (k <= 1) ? 1 : k * Inner(k - 1);
}

15) using: Namespaces & Disposal

using System.Text; // import namespace
using var reader = new StreamReader("file.txt"); // dispose at scope end

16) Ranges & Indices

var arr = new[]{10,20,30,40,50};
Console.WriteLine(arr[^1]);  // 50 (last)
var slice = arr[1..4];       // {20,30,40}

17) Nullable Reference Types

#nullable enable
string? name = null;
Console.WriteLine(name.Length); // warning: possible null

Turn on in project files: <Nullable>enable</Nullable>.

18) Coding Style Checklist

  • Naming: PascalCase for types/methods/properties; camelCase for locals/params.
  • Braces: Always use braces for clarity and safer diffs.
  • var: Prefer when the type is obvious. Otherwise spell the type.
  • Null safety: Use ?., ??, ??=. Avoid null unless intended.
  • Separation: One public class per file. Keep Program.cs tidy with local functions.
  • Culture: Use invariant parsing/formatting for CLI tools.

19) Mini-Challenges

Calculator 2.0 (switch expression + tuples)

public static (double result, bool ok) Calc(double a, double b, string op) =>
  op switch {
    "+" => (a + b, true),
    "-" => (a - b, true),
    "*" => (a * b, true),
    "/" when b != 0 => (a / b, true),
    _ => (0, false)
  };

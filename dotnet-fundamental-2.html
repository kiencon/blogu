2) Modern Pattern Matching

Switch expression

string Category(int n) => n switch
{
    < 0   => "negative",
    0     => "zero",
    <= 10 => "small",
    _     => "large"
};

Relational & logical patterns

string Grade(int s) => s switch
{
    < 0 or > 100 => "Invalid",
    >= 90        => "A",
    >= 80        => "B",
    >= 70        => "C",
    >= 60        => "D",
    _            => "F"
};

Type patterns

object o = "hello";
string r = o switch
{
    int i    => $"int {i}",
    string s => $"string {s}",
    null     => "null",
    _        => "other"
};

Tuple patterns

string Move((int x, int y) p) => p switch
{
    (0, 0)     => "origin",
    (0, var y) => $"vertical {y}",
    (var x, 0) => $"horizontal {x}",
    _          => $"point {p.x},{p.y}"
};

Property patterns

bool IsTeen(Person p) => p switch
{
    { Age: >= 13 and <= 19 } => true,
    _ => false
};

3) Expression-based Flow

string Access(int age) => age >= 18 ? "Adult" : "Restricted";

if (user is null) return;
if (!user.Active) return;

var labels = nums.Select(n => n switch
{
    < 0 => "neg",
    0   => "zero",
    _   => "pos"
});

4) Iteration Extras

foreach (var (val, idx) in items.Select((v, i) => (v, i)))
    Console.WriteLine($"{idx}: {val}");

        if (i % 2 == 0) yield return i;

5) Best Practices

  • Use control flow for expected/common paths.
  • Use exceptions for rare/exceptional cases.
  • Prefer early returns over deep nesting.
  • Favor switch expressions for mapping input → result.
  • Always handle all cases (_ or default).
